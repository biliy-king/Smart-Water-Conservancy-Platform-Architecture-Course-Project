COURSE DESIGN TASK

智慧水利监测与
管理平台开发

课程设计任务及指导书

技术栈：Cesium / Spring Boot / Vue.js

时间安排：第19-20周

01. 课程设计概要

全流程开发实战

不仅是写代码，而是按照标准的软件工程流程，完成从需求分析到系统交付的闭环。

核心目标：构建一套用于展示水工结构模型、倾斜摄影模型及监测数据的前后端平台。

前端技术

Vue.js + Cesium 三维可视化

后端技术

Spring Boot / Flask / FastAPI

能力培养

架构设计 / 团队协作 / 文档编写

02. 核心任务详情 (1-3)

1. 需求分析

  • 调研市场与现有解决方案
  • 明确用户角色（政府/工程师）
  • 撰写《需求分析文档》

2. 详细设计

  • 架构设计：前后端分离图
  • 数据库：E-R图，监测数据表
  • 接口设计：API 定义

3. 系统开发

  • Vue.js：模型展示与图表
  • Spring Boot：RESTful API
  • Cesium：三维结构加载

02. 核心任务详情 (4-5)

系统集成 (难点)

  • 集成前后端接口
  • 核心交互：实现 Cesium 三维模型与 ECharts 监测数据的实时绑定与交互。
  • 点击模型构件 -> 显示对应监测数据。

测试与演示

  • 编写测试用例（功能测试/集成测试）
  • 性能优化（模型加载速度等）
  • 项目演示：编写报告 + PPT 答辩

03. 前端可视化要求

前端界面需包含以下两大核心模块：

模型展示模块

（至少二选一）

A. 倾斜摄影模型 B. 水工三维结构模型

数据展示模块

（必须包含至少一种）

  • 数据表格 (Table)
  • 动态折线图 (Line Chart)
  • 动态增加测点测值 (如：2025-1-15, 0.21mm)

06. 组队与提交清单

分组规则

  • 每组最多 3人。
  • 组内需明确分工（前端/后端/文档/测试）。
  • 剩余同学随机分组。
  • 原则：小组优秀，组员才能全部优秀。

最终提交物

  1. 需求分析文档
  2. 详细设计文档 (含E-R图)
  3. 完整源代码 (前后端)
  4. 项目演示 PPT
